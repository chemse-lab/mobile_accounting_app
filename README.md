# تطبيق المحاسبة - البائع المتنقل 📱

تطبيق Flutter كامل لإدارة فرعين من السلع (وعدة أنواع لكل فرع)، مع مبيعات، مخزون،
مصاريف، ديون زبائن، تقارير، فواتير PDF، وطباعة عبر طابعة بلوتوث حرارية.

## ✅ خطوة أولى وإلزامية قبل أي شيء

بعض ملفات Gradle الثنائية (مثل `gradlew.bat` و `gradle-wrapper.jar`) لا يمكن إنشاؤها
كنص عادي. بعد فك ضغط/نسخ هذا المشروع على جهازك، افتح Terminal داخل مجلد المشروع
(`mobile_accounting_app/`) ونفّذ:

```bash
flutter create .
```

هذا الأمر **لن يحذف أو يستبدل** أي من الأكواد التي كتبناها (lib/, pubspec.yaml,
android/app/src/main/...) لأن Flutter يتجاهل الملفات الموجودة مسبقاً تلقائياً —
هو فقط يكمل الملفات الناقصة (gradlew، أيقونات افتراضية، .metadata، إلخ).

ثم:

```bash
flutter pub get
```

## 📱 التشغيل على جهاز Android

1. فعّل "خيارات المطور" و "تصحيح USB" على جوالك، ووصله بالكمبيوتر بكابل USB
   (أو استخدم محاكي Android Studio).
2. تأكد من الاتصال:
   ```bash
   flutter devices
   ```
3. شغّل التطبيق:
   ```bash
   flutter run
   ```

## 🖨️ لربط طابعة البلوتوث

1. اذهب لإعدادات البلوتوث في نظام أندرويد نفسه وقم بإقران (Pair) الطابعة
   الحرارية أولاً (نفس خطوة ربط أي جهاز بلوتوث عادي).
2. افتح التطبيق → القائمة الجانبية (☰) → الإعدادات → "طابعة الإيصالات" →
   اضغط "عرض الأجهزة المقترنة" → اختر طابعتك → اتصال.
3. من الآن فصاعداً، أي عملية بيع جديدة ستعرض زر "طباعة بلوتوث" يطبع
   إيصالاً مباشرة (حجم 58mm افتراضياً، يمكن تعديله لـ 80mm من الكود في
   `lib/services/bluetooth_print_service.dart`).

## 📦 بناء ملف APK للتثبيت أو النشر

```bash
flutter build apk --release
```

الملف الناتج: `build/app/outputs/flutter-apk/app-release.apk`

⚠️ **قبل النشر على Google Play**: ملف `android/app/build.gradle` حالياً يستخدم
مفتاح التوقيع الافتراضي (`debug`) لتسهيل التجربة. يجب توليد مفتاح توقيع حقيقي
(`keystore`) واستبداله في `signingConfigs` قبل أي نشر رسمي.

## 🎨 الأيقونة واسم التطبيق

- اسم التطبيق الظاهر: `android/app/src/main/AndroidManifest.xml` → `android:label`
- الأيقونة: استبدل ملفات `android/app/src/main/res/mipmap-*/ic_launcher.png`
  (تتولد افتراضياً بأمر `flutter create .`)، أو استخدم حزمة
  [`flutter_launcher_icons`](https://pub.dev/packages/flutter_launcher_icons)
  لتوليدها أوتوماتيكياً من صورة واحدة.

## 🗂️ هيكلة المشروع

```
lib/
  models/        نماذج البيانات (Branch, ProductType, Product, Sale, Expense, Customer)
  db/            قاعدة بيانات SQLite (تعمل على Android و سطح المكتب بنفس الكود)
  services/      حالة التطبيق (Provider)، الطباعة بلوتوث، فواتير PDF، النسخ الاحتياطي
  screens/       كل شاشات التطبيق
  widgets/       عناصر واجهة قابلة لإعادة الاستخدام
  utils/         تنسيق العملة والتاريخ
android/         ملفات مشروع Android الجاهزة (أذونات البلوتوث مُفعّلة بالكامل)
```

## 🔜 الخطوة التالية: نسخة سطح المكتب

بما أن قاعدة البيانات مبنية بـ `sqflite_common_ffi` (تعمل على Windows/Linux/macOS
أيضاً)، أغلب الكود في `lib/` قابل لإعادة الاستخدام مباشرة لتطبيق سطح مكتب —
فقط نحتاج تفعيل `flutter create . --platforms=windows,linux,macos` وتعديل بعض
الشاشات (مثلاً تعطيل تبويب البلوتوث) لتناسب شاشة أكبر.
